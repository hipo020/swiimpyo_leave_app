# 쉼표 최종 점검본 적용 방법

## 1. Supabase 보안 설정

이미 Supabase SQL Editor에서 `supabase_security.sql`을 실행하고 RLS·정책·제약조건 확인까지 마쳤다면 다시 실행할 필요는 없어.
새 Supabase 프로젝트에 옮길 때만 SQL을 한 번 적용하면 돼.

## 2. 웹 파일 교체

`index.html`, `styles.css`, `app.js`를 같은 폴더에 올려. 세 파일의 상대 경로를 유지해야 해.

## 이번 최종 점검에서 추가 반영된 내용

- 로그인 화면의 개발자용 RLS 안내 문구 삭제
- 로그인·동기화 오류 문구에서 일반 사용자에게 불필요한 기술 용어 정리
- 로그인 직후 인증 이벤트가 겹쳐도 같은 사용자 데이터를 중복 조회하지 않도록 잠금 추가
- 계정 전환 도중 이전 계정의 로딩 결과가 화면을 덮어쓰지 않도록 방어
- 남아 있던 이전 추천 UI 로직이 없는 요소를 참조해도 오류가 나지 않도록 방어
- 모든 버튼에 `type="button"`을 명시해 추후 폼 구조가 추가되어도 의도치 않은 제출을 방지
- 기존 모바일 개선, XSS 방어, 중복 등록 방지, 미저장 경고, 삭제 되돌리기, 인증 감지, 오프라인 안내 유지

## 관리자 이메일 변경

관리자 계정을 바꿀 때는 `app.js`의 `HOLIDAY_ADMIN_EMAIL`과 `supabase_security.sql`의 이메일 값을 둘 다 같은 주소로 바꿔야 해.

## 이번 모바일 하단 정리
- 하단 메뉴 때문에 마지막 콘텐츠가 가려지지 않도록 여백을 확보했습니다.
- 가운데 `+` 버튼을 하단 메뉴 안쪽 중앙에 맞췄습니다.
- 달력 칸 높이와 카드 하단 여백, 메뉴 그림자를 정돈했습니다.


## 이번 모바일 하단바 정리
- 가운데 + 버튼을 하단바 내부 버튼으로 변경
- 하단바 반투명 겹침 제거
- 하단 콘텐츠가 비쳐 보이지 않도록 배경 마스크 추가
- 모바일에서는 기존 플로팅 + 버튼을 숨김


## 모바일 하단 UI
- 하단 고정 바에는 `캘린더`, `연차 리포트` 두 탭만 표시됩니다.
- 새 연차 등록 `+` 버튼은 하단 바 위 오른쪽에 독립된 플로팅 버튼으로 표시됩니다.
