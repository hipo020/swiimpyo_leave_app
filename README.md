# 쉼표 안전성 강화본 적용 방법

## 1. 먼저 SQL 적용

Supabase Dashboard → SQL Editor에서 `쉼표_Supabase_RLS_보안설정.sql` 내용을 실행해.
이 단계가 실제 사용자별 데이터 보호와 공휴일 관리자 권한을 담당해.

실행 중 `unique index` 또는 `constraint` 오류가 나면 기존 중복·잘못된 데이터가 있다는 뜻이야. 오류 메시지와 함께 데이터를 먼저 정리한 뒤 다시 실행하면 돼.

## 2. 웹 파일 교체

단일 파일을 쓰는 경우 `쉼표_안전성강화_최종본.html`을 기존 HTML 대신 배포하면 돼.

분리형 패키지를 쓰는 경우 `index.html`, `styles.css`, `app.js`를 같은 폴더에 올려. 세 파일의 상대 경로를 유지해야 해.

## 반영된 기능

- 같은 등록을 재시도해도 중복 저장되지 않는 고정 ID + upsert
- 삭제 후 8초 동안 되돌리기
- 입력 중 창을 닫을 때 미저장 내용 경고
- 모바일 브라우저 뒤로가기 시 입력창 닫기 처리
- 로그인·로그아웃·토큰 갱신 상태 감지
- 오프라인 상태 표시와 입력 유지
- 사용자 입력 HTML 이스케이프와 길이 검증
- CSV 수식 주입 방어
- OAuth 로그인 복귀 경로 보존
- Supabase 스크립트 버전 고정 및 잘못된 script 구조 분리
- RLS, DB 제약조건, 최소 권한 SQL 제공

## 관리자 이메일 변경

관리자 계정을 바꿀 때는 HTML/JS의 `HOLIDAY_ADMIN_EMAIL`과 SQL의 이메일 값을 둘 다 같은 주소로 바꿔야 해.


## 모바일 사용성 개선

- 하단 탭과 겹치지 않는 토스트 위치
- 리포트 탭 선택 시 리포트 영역 자동 이동
- 모바일 수정·삭제 버튼 터치 영역 확대
- 플로팅 등록 버튼을 하단 탭 위로 이동
- 모바일 키보드 표시 시 바텀시트 높이 보정
